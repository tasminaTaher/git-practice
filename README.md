# git-practice
This exercise was aimed at practicing the basic Git and GitHub workflow, using a simple C++ program (main.cpp) that displays student information.
I started by creating  a newa public GitHub repository named git-practice. Within this repository, I created an initial file called main. cpp using the GitHub interface, which included a C++ program designed to display student information such as  name, student ID, section, and another personal field., and I committed this with the note "Initial commit. "
Subsequently, I launched a terminal and duplicated the repository to my local computer through git clone, then navigated into the project directory using cd git-practice. To ensure that everything was in order right after the cloning process, I executed git status, confirming the local copy was identical to the remote repository.
I proceeded to open main. cpp in my code editor and modified the variables for name, ID, and section to reflect my personal details, in addition to including another field of my choice. After making these changes and saving the file, I utilized git diff to examine the specific modifications I had implemented, then checked git status again, which indicated that the file had been altered.
To prepare the changes for committing, I staged the modifications by executing git add main. cpp and verified the staging process through another git status command. Following this, I committed the staged alterations locally with a detailed message using git commit -m "Updated student information. " Ultimately, I pushed the commit to GitHub using git push and refreshed the repository page in my web browser to verify that the updated version of main. cpp had been uploaded successfully.
This exercise gave me a better understanding of how Git keeps track of changes and how local work is synchronized with a remote GitHub repository. It also helped me become more comfortable with the basic Git workflow.
